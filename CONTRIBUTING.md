# Contributing to DataPulse

Thank you for your interest in contributing to DataPulse! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Datapulse.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Set up your development environment (see README.md)

## Development Workflow

1. Make your changes in your feature branch
2. Write or update tests for your changes
3. Run the test suite: `python -m pytest`
4. Ensure code follows project style guidelines
5. Commit your changes with clear, descriptive messages
6. Push to your fork and submit a pull request

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions focused and modular
- Comment complex logic

## Testing

- Write unit tests for new features
- Ensure all tests pass before submitting PR
- Aim for good test coverage
- Test both success and error cases

## Pull Request Process

1. Update documentation if needed
2. Add a clear description of your changes
3. Reference any related issues
4. Wait for review and address feedback
5. Once approved, your PR will be merged

## Reporting Bugs

- Use GitHub Issues
- Include clear title and description
- Provide steps to reproduce
- Include error messages and logs
- Mention your environment (OS, Python version, etc.)

## Feature Requests

- Open a GitHub Issue
- Describe the feature and its benefits
- Discuss implementation approach if possible

## Questions?

Feel free to open a discussion or issue for any questions!
